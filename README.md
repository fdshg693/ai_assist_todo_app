## 現状のコード状況

### 📦 現在のプロトタイプについて

このリポジトリには、フロントエンドとバックエンドの基本的な通信を確認するためのプロトタイプが既に実装されています。

**技術スタック:**
- **バックエンド**: .NET 10.0 Web API
  - ASP.NET Core を使用した RESTful API
  - `backend/` ディレクトリに配置
  - デフォルトでポート 5000 番で起動
  
- **フロントエンド**: Vite + React + TypeScript
  - モダンなビルドツール Vite による高速な開発環境
  - `frontend/` ディレクトリに配置
  - デフォルトでポート 5173 番で起動

**プロトタイプの内容:**
- バックエンドとフロントエンド間の基本的な HTTP 通信の動作確認
- 開発サーバーの起動と動作確認が可能な最小限の構成
- これから本格的な機能を追加していくためのベースとなるコード

### 🚀 開発の進め方

初めてこのプロジェクトに参加される方は、以下のステップで進めてください:

1. **環境構築とプロトタイプの動作確認**
   - まずは [docs/QUICKSTART.md](docs/QUICKSTART.md) を参照して、開発環境を構築してください
   - バックエンドとフロントエンドの両方を起動し、正常に動作することを確認してください

2. **ブランチ戦略**
   - 新しい機能を開発する際は、必ず `main` ブランチから新しいブランチを作成してください
   - ブランチ名は機能や目的が分かりやすい名前にしてください（例: `feature/add-authentication`, `learning/try-zustand`）

3. **学習優先の開発スタイル**
   - このプロジェクトでは、**様々なライブラリやフレームワークを試しながら学習すること**を最優先としています
   - 完璧な機能実装よりも、新しい技術を試して理解を深めることに重点を置いています
   - 実験的なコードや試行錯誤のプロセスを大切にしてください

4. **段階的な機能開発**
   - 本格的なTodo管理機能の実装は、基本的な技術スタックに慣れてから進めます
   - まずはこのプロトタイプを基に、様々なライブラリや機能追加を試してみてください
   - 慣れてきたところで、小さな機能やUI コンポーネントから実装を始めていきましょう

## 🎯 直近実装な主な機能

### 現在実装予定の機能

- **基本的なTodo管理**
  - タスクの作成・編集・削除
  - タスクの完了状態管理
  - タスクの優先順位設定

- **AIレコメンド機能**
  - 次に取り組むべきTodoの推薦
  - タスクの優先順位付けサポート
  - 作業効率を最適化する提案

- **AIサマリ機能**
  - 現在のタスク状況の要約
  - 進捗状況の可視化
  - タスク完了率の分析

### 将来実装予定の機能

- **AIによるタスク分解**
  - 大きなTodoを実行可能なサブタスクに自動分割
  - タスクの依存関係の提案

- **学習リソース生成**
  - Todoに関連する学習リソースの自動提案
  - スキルアップのための参考情報の提供
